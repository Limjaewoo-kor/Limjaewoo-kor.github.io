    • 제약 분야 종사자들이 해당 화면을 주기적으로 크롤링하여 서버에 큰 부하 발생.

      • 바로가기를 통해 접속 시 바코드를 인식하는 카메라 화면 제공.
    • ActiveX 폐지 이후 사용자 동의 없이 바로가기 생성 불가.

    Solution

    2 solutions

  • 해결 방법:
    • "어떻게 부하를 줄일 수 있을까?"라는 고민 끝에 View의 캐시화를 통해 해결.
    • Filter에서 URL로 분기를 태워 특정 화면을 파일로 캐시화.
    • ContentCachingResponseWrapper를 활용해 랜더링된 HTML 파일을 WAS 내 특정 경로에 저장.
    • 데이터가 변경되지 않은 경우에는 저장된 파일을 사용해 응답 시간과 서버 부하 감소.
  • 성과:
    • 캐시를 통한 안정화로 서버 부하 감소 및 응답 시간 단축.
    • 컨테이너 다운 문제 해결 및 TableSpace 용량 초과 방지.
  • 해결 방법:
    • PWA (Progressive Web App)의 a2hs (Add to Home Screen) 기능 활용:
      • 사용자 동의를 통해 웹 사이트 바로가기 아이콘 생성.
    • PWA 구현 절차:
      • 1. 메니페스트 파일 작성.
      • 2. HTTPS 지원 및 로컬호스트 설정.
      • 3. Service Worker 등록.
    • 메인 화면 진입 시 카메라 호출 함수를 추가해 바코드 인식 기능 제공.
  • 성과:
