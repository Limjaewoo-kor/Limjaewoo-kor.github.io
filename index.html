j_universe_AI (2025.04 ~ )

프로젝트 개요

Overview
  • j-universe 사이트 바로가기
  • Github 소스 바로가기

j_universe_AI는 LLM 기술을 활용하여 다양한 실생활 문제를 해결하는 웹 애플리케이션 프로젝트 모음입니다.
모든 프로젝트는 FastAPI 기반 백엔드와 React 기반 프론트엔드로 구성되었으며, PostgreSQL DB 및 LangChain 기반의 AI 기능을 중심으로 개발되었습니다.

Currently in progress

주요 기능

  • 1. Malhaejo (말해조)
    • 사용자가 핵심 내용을 입력하면, 정중하고 공손한 비즈니스 문장 또는 민원 문서를 자동으로 생성해주는 AI 문장 생성 도우미입니다.
    • GPT 기반 자연어 생성 기술을 활용하여 한국어의 높임말, 맞춤법, 상황별 표현 등을 정교하게 적용합니다.

  • 2. Rumble Guide Chatbot
    • 디지몬 슈퍼럼블 게임의 가이드를 위한 LLM 기반 챗봇으로, OpenAI API + LangChain + RAG + Chroma를 활용하여 캐릭터 정보, 공략, 추천 세팅 등의 질문에 실시간 대응합니다.

  • 3. 계산해 (말로 계산하는 AI 계산기)
    • 사용자가 자연어로 입력한 계산 요청(예: "23500원을 30% 할인한 금액을 알려줘")을 이해하고 숫자로 변환하여 계산해주는 AI 계산기입니다.
    • 기본 계산 외에도 이자 계산기, 환율 계산기, 단위 변환기 등 다양한 기능을 탭 형태로 분리하여 제공합니다.

공통 기술 구성

  • FastAPI + LangChain 백엔드 / React 프론트엔드 / PostgreSQL DB
  • 모바일 대응(반응형 CSS), PWA 지원(웹앱 설치 기능)
  • Google Analytics 연동
  • Vercel(front) / Render(back) / Neon(DB)로 서비스 배포

Issue

파라프레이즈(표현 다양성) 문제
  • 초기에는 유사한 의미의 다양한 질문 표현에 대해 LLM이 일관되지 못한 응답을 제공하는 문제가 있었습니다.

Solution

LLM Fine-Tuning 없이 해결한 전략 3가지
  • Few-shot Prompting: 다양한 예시 문장을 포함한 프롬프트 제공
  • Chunk 조정: chunk_size 및 chunk_overlap 최적화로 문맥 유지
  • Retriever 개선: Top-k 값 조정 및 relevance filter 적용

2가지 주요 이슈 해결
  • 1. 대용량 조인 쿼리로 인한 서버 부하 및 컨테이너 장애
    • 약 20만 건의 물품 데이터를 대상으로 15개 이상의 테이블 조인 쿼리 실행 시, 응답 시간이 최대 60초 이상 소요
    • 외부 기관 및 제약사에서 해당 화면을 주기적으로 크롤링하여 서버 부하 가중
    • 결과적으로 TableSpace 용량 초과, 컨테이너 다운 등의 장애가 반복 발생

  • 2. 시각 장애인을 위한 모바일 접근성 개선 요구
    • 모바일에서 웹사이트 바로가기를 생성하고 접속 시 카메라 화면에서 바코드 인식 기능 제공 필요
    • ActiveX 폐지로 인해 기존처럼 사용자 동의 없이 바로가기 생성 불가능

Solution

문제 해결을 위한 주요 전략
  • 1. 서버 부하 및 컨테이너 다운 문제 해결
    • View 캐싱 도입: 특정 URL 진입 시 Filter를 통해 ContentCachingResponseWrapper로 HTML 렌더링 결과를 저장
    • 파일 캐시화 전략: WAS 내 정적 경로에 HTML 파일을 저장하고, 동일 요청 시 캐시 파일을 응답
    • 효과: 데이터가 변경되지 않을 경우 즉시 응답 가능 → 응답 속도 향상 및 서버 부하 감소

  • 2. 모바일 웹 접근성 및 바로가기 생성 문제 해결
    • PWA (Progressive Web App) 도입으로 a2hs(Add to Home Screen) 기능 구현
    • PWA 구현 절차:
      • 메니페스트 파일 작성 및 HTTPS 환경 설정
      • Service Worker 등록을 통한 오프라인 대응 및 푸시 기반 구조 구현
    • 카메라 호출 기능 추가: 메인 화면 진입 시 바코드 인식을 위한 카메라 함수 트리거

Result

개선 성과
  • 서버 안정화: View 캐싱으로 응답 속도 최대 80% 단축 및 컨테이너 다운 방지
  • 접근성 향상: 상위 기관의 요구사항 충족 및 시각 장애인을 위한 UX 강화
  • 모바일 UX 개선: 바로가기 아이콘 + 카메라 기능을 통해 PWA 기반 사용자 경험 개선

프로젝트 개요

Overview

실제 보드게임 카페를 운영하는 관리자의 시나리오를 상상하여 구현한 관리자용 PC 웹 애플리케이션입니다. 회원/게임/음식에 대한 CRUD 기능 및 사용자 주문/게임 실행 기능을 포함하며, 로그인 기반의 세션 관리, 인터셉터, 필터 등을 통해 구조화된 백엔드 흐름을 학습하고자 하였습니다.

기술 스택

  • 백엔드: Spring Boot, MyBatis(Java Config), Oracle DB
  • 프론트: Thymeleaf
  • 기타: LoginSession, Filter, Interceptor

주요 기능

  • 관리자 기능
    • 회원 관리: 회원가입, 회원 목록 조회
    • 게임 관리: 게임 추가, 게임 목록 조회
    • 음식 관리: 음식 추가, 음식 목록 조회

  • 사용자 기능
    • 게임 이용: 게임 시작
    • 주문 기능: 음식 주문, 주문 리스트 조회

Issue

MyBatis 설정 변경 관련 이슈
  • 문제 발생 배경: 기존 XML 기반 설정에서 Java Config 기반 설정으로 전환 중 발생
  • 증상: SQL 실행 로그가 출력되지 않음 → 디버깅 및 개발 생산성 저하

Solution

Java Config 기반 SQL 로깅 설정
  • MyBatis 설정: dataSource, SqlSessionFactory, MapperScan 등 전반적인 설정을 Java Config 방식으로 재구성
  • 로깅 문제 해결: RefreshableSqlSessionFactoryBean 내부 로직을 분석하고, SQL 로그 출력 설정을 코드 기반에서 명시적으로 지정
  • 성과: SQL 로깅 정상화 및 설정 충돌 없이 안정적인 Java Config 기반 프로젝트로 전환 완료

프로젝트 개요

Overview
  • Github 소스 바로가기

이 프로젝트는 키워드 중심의 콘텐츠 크롤링, 트렌드 분석, AI 자동 글 생성을 통합한 콘텐츠 생성 및 분석 자동화 도구입니다. 자연어 처리 기술(NLP)과 트렌드 시각화를 결합하여 블로거 및 마케팅 초안 작성자를 위한 자동화 지원 도구를 지향합니다.

기술 스택

  • 크롤링: 네이버 검색 API, Selenium
  • 요약: KoBART (digit82/kobart-summarization)
  • 키워드 추출: KeyBERT (model="all-MiniLM-L6-v2")
  • AI 생성: KoBART, LGAI-EXAONE, KoGPT2, KANANA
  • 트렌드 분석: 네이버 데이터랩 API

주요 기능

  • 1. 블로그 크롤링
    • 네이버 검색 API 및 Selenium을 활용하여 특정 키워드 관련 상위 블로그 포스팅 3~5개 수집
    • 저장 항목: 제목, 검색 키워드, 본문 내용, 요약(자동화), 중요 키워드
    • 요약: KoBART 기반 자동 요약, 키워드: KeyBERT 기반 추출

  • 2. AI 문장 생성
    • 저장된 포스팅 상세 페이지에서 AI 언어 모델을 사용한 글 생성 기능 제공
    • 사용 모델: KoBART, LGAI-EXAONE, KoGPT2, KANANA (4가지 모델 비교 가능)
    • 복사 및 편집 기능 제공으로 사용자의 후속 작업을 지원

  • 3. 키워드 트렌드 분석
    • 네이버 데이터랩 API를 통해 연/월별 트렌드 그래프 시각화
    • 다중 키워드 입력 후 비교 분석 가능

Issue

하드웨어 자원 문제
  • 여러 개의 언어 모델을 사용할 때 CPU / GPU / Memory 리소스 부족 문제가 발생

Solution

하드웨어 리소스 최적화 전략
  • 1. 디바이스 자동 할당 설정:
    • device_map="auto" 및 ("cuda" if torch.cuda.is_available() else "cpu") 설정으로 GPU 자동 할당
  • 2. 메모리 최적화:
    • torch.cuda.init() 및 torch.cuda.empty_cache()를 활용하여 GPU 초기화 및 캐시 메모리 정리
