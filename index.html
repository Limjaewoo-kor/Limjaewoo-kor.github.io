j_universe_AI (2025.04 ~ )

프로젝트 개요

Overview
  • J-universe 사이트 바로가기

j_universe_AI는 LLM 기술을 활용하여 다양한 실생활 문제를 해결하는 웹 애플리케이션 프로젝트 모음입니다.
모든 프로젝트는 FastAPI 기반 백엔드와 React 기반 프론트엔드로 구성되었으며, PostgreSQL DB 및 LangChain 기반의 AI 기능을 중심으로 개발되었습니다.

Currently in progress

주요 기능

  • 1. Malhaejo (말해조)
    • 사용자가 핵심 내용을 입력하면, 정중하고 공손한 비즈니스 문장 또는 민원 문서를 자동으로 생성해주는 AI 문장 생성 도우미입니다.
    • GPT 기반 자연어 생성 기술을 활용하여 한국어의 높임말, 맞춤법, 상황별 표현 등을 정교하게 적용합니다.

  • 2. Rumble Guide Chatbot
    • 디지몬 슈퍼럼블 게임의 가이드를 위한 LLM 기반 챗봇으로, OpenAI API + LangChain + RAG + Chroma를 활용하여 캐릭터 정보, 공략, 추천 세팅 등의 질문에 실시간 대응합니다.

  • 3. 계산해 (말로 계산하는 AI 계산기)
    • 사용자가 자연어로 입력한 계산 요청(예: "23500원을 30% 할인한 금액을 알려줘")을 이해하고 숫자로 변환하여 계산해주는 AI 계산기입니다.
    • 기본 계산 외에도 이자 계산기, 환율 계산기, 단위 변환기 등 다양한 기능을 탭 형태로 분리하여 제공합니다.

  • 4. Brain Trainer (두뇌트레이너)
    • 연령대별 기억력·집중력 향상을 위한 퀴즈 서비스로, 상식·산수·이야기 기억력·사자성어 등 다양한 유형의 문제를 제공합니다.
    • 기존 DB 문제와 LLM 기반 문제를 혼합 운영하여 문제 다양성과 난이도 조절 기능을 구현하였습니다.
    • CrewAI 멀티에이전트 구조를 관리자 모드에 적용하여 문제 생성 품질을 향상시켰습니다.
      • Generator Agent: 난이도·유형 조건에 맞춰 문제 생성
      • Validator Agent: 정답·형식·난이도 일치 여부 검증
      • Formatter Agent: JSON 포맷 변환
    • 생성 결과 및 처리 시간을 로그 파일로 저장하여 품질 모니터링 체계를 구축하였습니다.

공통 기술 구성

  • FastAPI + LangChain 백엔드 / React 프론트엔드 / PostgreSQL DB
  • 모바일 대응(반응형 CSS), PWA 적용(웹앱 설치 기능)
  • Google Analytics 연동
  • Vercel[ front ] / Render -> Oracle Cloud Infrastructure(OCI)[ back ] / Neon[ DB ]로 서비스 배포

Issue

파라프레이즈(표현 다양성) 문제
  • 초기에는 유사한 의미의 다양한 질문 표현에 대해 LLM이 일관되지 못한 응답을 제공하는 문제가 있었습니다.
백엔드 인프라 한계 및 비용 문제
  • Render 무료 플랜에서 빈번한 슬립 현상 및 속도 지연, 리소스 제약으로 인해 실사용자 경험에 불편이 발생했습니다.
LLM 호출 비용 문제 (두뇌트레이너)
  • 두뇌트레이너의 퀴즈 문제를 전부 LLM으로 생성할 경우, 문제당 API 호출 비용이 누적되어 운영 부담이 발생했습니다.

Solution

LLM Fine-Tuning 없이 해결한 전략 3가지
  • Few-shot Prompting: 다양한 예시 문장을 포함한 프롬프트 제공
  • Chunk 조정: chunk_size 및 chunk_overlap 최적화로 문맥 유지
  • Retriever 개선: Top-k 값 조정 및 relevance filter 적용
인프라 개선 전략
  • 백엔드 이전: Render 무료 플랜에서 발생하던 성능/슬립 이슈를 해결하기 위해 Oracle Cloud Infrastructure(OCI)의 Compute 인스턴스로 직접 배포 전환
  • 배포 방식 개선: DevOps 파이프라인 대신 수동 `.zip` 빌드 및 SSH 기반 직접 배포 방식 채택 → 배포 시간 단축 및 문제 발생 시 디버깅 용이
  • 도메인 연동 및 HTTPS 인증서 설정: Nginx 리버스 프록시 + Let's Encrypt 적용으로 사용자 도메인 환경에서도 안정적인 서비스 제공
LLM 호출 비용 최적화 (두뇌트레이너)
  • 기존 DB 문제와 LLM이 생성한 신규 문제를 혼합하여 출제하는 구조를 도입
  • 혼합 운영으로 LLM 호출 횟수를 절반 이하로 줄이며, 문제 다양성과 품질을 유지

  • 1. Nexacro N 솔루션 전환 (메인 개발자)
    • 코드 호환성 점검: 변경된 API 목록 정리 및 호환성 검토
    • UI/UX 최적화: 최신 Nexacro N 스타일 및 컴포넌트 재설계
    • 성과: 전체 시스템 응답 속도 향상 및 유지보수 용이성 증가

  • 2. 웹 접근성 및 보안 조치 (메인 개발자)
    • 웹 접근성 개선: WCAG 2.1 기준 충족 및 ARIA 적용
    • 보안 강화: CSRF 토큰 적용, 필터 기반 SQL Injection 방어
    • 성과: 보안 인증 및 웹 접근성 인증 통과 가능 수준 확보

  • 3. 장기추적시스템 고도화 (서브 개발자)
    • DB 성능 개선: 인덱스 추가 및 쿼리 튜닝
    • 모듈화 작업: DAO/Service 분리 및 기능 리팩토링
    • 성과: 처리 속도 향상 및 유지보수 용이성 강화

  • 4. JEUS → JBoss / WebtoB → Apache 전환 (메인 개발자)
    • 환경 설정: JBoss standalone.xml, Apache httpd.conf 커스터마이징
    • 세션/보안 최적화: Infinispan 세션 클러스터링, SSL 및 mod_security 적용
    • 성과: 오픈소스 도입으로 인한 라이선스 비용 절감 및 보안성 강화

Managed_Web (2021.01 ~ 2023.05)

프로젝트 개요

Overview
  • 의약품 안전나라 사이트 바로가기

본 프로젝트는 전자정부 프레임워크 + Nexacro(넥사크로) + Thymeleaf 기반으로 운영되는 공공 의약품 정보 시스템입니다. 유지보수 및 기능개선 개발자로 참여하였으며, 대용량 데이터 처리와 접근성 문제 해결 등 여러 현업 요구사항을 해결해 나갔습니다.

Issue

2가지 주요 이슈 해결
  • 1. 대용량 조인 쿼리로 인한 서버 부하 및 컨테이너 장애
    • 약 20만 건의 물품 데이터를 대상으로 15개 이상의 테이블 조인 쿼리 실행 시, 응답 시간이 최대 60초 이상 소요
    • 외부 기관 및 제약사에서 해당 화면을 주기적으로 크롤링하여 서버 부하 가중
    • 결과적으로 TableSpace 용량 초과, 컨테이너 다운 등의 장애가 반복 발생

  • 2. 시각 장애인을 위한 모바일 접근성 개선 요구
    • 모바일에서 웹사이트 바로가기를 생성하고 접속 시 카메라 화면에서 바코드 인식 기능 제공 필요
    • ActiveX 폐지로 인해 기존처럼 사용자 동의 없이 바로가기 생성 불가능

Solution

문제 해결을 위한 주요 전략
  • 1. 서버 부하 및 컨테이너 다운 문제 해결
    • View 캐싱 도입: 특정 URL 진입 시 Filter를 통해 ContentCachingResponseWrapper로 HTML 렌더링 결과를 저장
    • 파일 캐시화 전략: WAS 내 정적 경로에 HTML 파일을 저장하고, 동일 요청 시 캐시 파일을 응답
    • 효과: 데이터가 변경되지 않을 경우 즉시 응답 가능 → 응답 속도 향상 및 서버 부하 감소

  • 2. 모바일 웹 접근성 및 바로가기 생성 문제 해결
    • PWA (Progressive Web App) 도입으로 a2hs(Add to Home Screen) 기능 구현
    • PWA 구현 절차:
      • 메니페스트 파일 작성 및 HTTPS 환경 설정
      • Service Worker 등록을 통한 오프라인 대응 및 푸시 기반 구조 구현
    • 카메라 호출 기능 추가: 메인 화면 진입 시 바코드 인식을 위한 카메라 함수 트리거

Result

개선 성과
  • 서버 안정화: View 캐싱으로 응답 속도 최대 80% 단축 및 컨테이너 다운 방지
  • 접근성 향상: 상위 기관의 요구사항 충족 및 시각 장애인을 위한 UX 강화
  • 모바일 UX 개선: 바로가기 아이콘 + 카메라 기능을 통해 PWA 기반 사용자 경험 개선
